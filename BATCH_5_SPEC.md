# TAX TJOMMIE v3 — BATCH 5 BUILD SPEC
## PDF Reports, Document Storage, Receipt OCR, Cross-Device Sync

---

## OVERVIEW

Batch 5 adds the four features that turn Tax Tjommie from a tracker into a complete tax preparation tool:

1. **PDF Year-End Report** — THE deliverable. "Send this to your tax lady."
2. **Document Storage** — Store receipts, IRP5s, certificates as proof.
3. **Receipt OCR** — Snap a receipt, auto-read amount + date + vendor.
4. **Cross-Device Sync** — Export/import data so phone and laptop stay in sync.

Build in this order. Each feature works independently.

## FEATURE 2: DOCUMENT STORAGE

### What it does
Users can upload and store tax-related documents (photos of receipts, PDFs of IRP5s, certificates) directly in the app. Documents are stored as base64 in localStorage.

### Document Types
| Type | Description | Expected Format |
|---|---|---|
| receipt | Receipt for an expense | Photo (JPG/PNG) |
| irp5 | IRP5 or IT3(a) from employer/fund | PDF or photo |
| medical_cert | Medical aid tax certificate | PDF or photo |
| section18a | Donation tax certificate | PDF or photo |
| logbook | Vehicle logbook records | PDF or photo |
| ra_cert | Retirement annuity tax certificate | PDF or photo |
| other | Any other supporting document | PDF or photo |

### Upload Flow
1. User taps "Add Document" (from Documents screen or from an expense)
2. Options:
   - 📸 **Take Photo** (mobile camera)
   - 📤 **Upload File** (file picker — accepts JPG, PNG, PDF)
3. After capture/upload:
   - **Document type** dropdown (receipt, IRP5, etc.)
   - **Description** free text ("Woolworths receipt March 2025")
   - **Date** (defaults to today)
   - **Link to expense** (optional — dropdown of recent unlinked expenses)
   - **Tax year** (auto from active year, can change)
4. Save → stored in localStorage as base64

### Storage Format
```json
{
  "documents": [
    {
      "id": "uuid",
      "type": "receipt",
      "description": "Woolworths office supplies",
      "date": "2025-03-15",
      "taxYear": "2025/26",
      "linkedExpenseId": "expense-uuid-123",
      "fileType": "image/jpeg",
      "fileName": "receipt_001.jpg",
      "fileData": "data:image/jpeg;base64,/9j/4AAQ...",
      "fileSize": 245000,
      "created": "2025-03-15T14:30:00Z"
    }
  ]
}
```

### Documents Screen (under More → Documents)
- Filter by: type, tax year, date range
- Grid view: thumbnail previews (for images) or PDF icon
- Each document shows: thumbnail, description, date, linked expense (if any)
- Tap to view full-size
- Delete with confirmation
- Count by type: "Receipts: 42 | IRP5s: 1 | Medical: 1 | Other: 3"

### Linking to Expenses
- When viewing an expense, show "📎 Attach document" if none linked
- When viewing a document, show "🔗 Linked to: [expense description]" if linked
- Expense list shows a 📎 icon next to expenses with documents

### Storage Warning
- Track total storage used: `JSON.stringify(data).length`
- At 3MB: show subtle warning "Storage: 3MB of ~5MB used"
- At 4MB: show amber warning "Storage getting full. Consider exporting your data."
- At 4.5MB: show red warning "Storage nearly full. Export data to free space. New documents may fail to save."
- Show storage indicator in Settings

### Desktop vs Mobile
- On mobile: "Take Photo" uses the camera directly (via `<input type="file" accept="image/*" capture="environment">`)
- On desktop: "Take Photo" option is hidden — only "Upload File" shown
- On desktop: drag-and-drop support for the upload area
- PDF viewing: on tap, open in new browser tab (create blob URL from base64)

## FEATURE 3: RECEIPT OCR

### What it does
When a user takes a photo of a receipt, the app automatically reads the total amount, date, and vendor name using Tesseract.js. The user confirms or corrects the values. This makes expense logging from receipts take seconds instead of minutes.

### Library
**Tesseract.js** loaded on-demand (only when OCR is triggered):
```html
<!-- Load only when needed, not on page load -->
<script>
function loadTesseract() {
  return new Promise((resolve) => {
    if (window.Tesseract) { resolve(); return; }
    const script = document.createElement('script');
    script.src = 'https://cdn.jsdelivr.net/npm/tesseract.js@5/dist/tesseract.min.js';
    script.onload = resolve;
    document.head.appendChild(script);
  });
}
</script>
```

### OCR Flow
1. User taps 📸 on the Log Expense screen
2. Camera opens (or file picker on desktop)
3. Photo captured → show preview with "Scanning receipt..." spinner
4. Tesseract.js processes the image (English language)
5. App parses the OCR text looking for:
   - **Total amount**: Look for patterns like "TOTAL R123.45", "AMOUNT DUE R123.45", "BALANCE R123.45", "TOT R123", or the largest Rand amount on the receipt
   - **Date**: Look for date patterns (DD/MM/YYYY, DD-MM-YYYY, DD MMM YYYY, etc.)
   - **Vendor/store name**: Usually the first line(s) of text, or look for common SA retailers
6. Show extracted values in editable fields:
   ```
   We found:
   Amount:  R 245.90  [edit]
   Date:    15/03/2025 [edit]  
   Store:   Woolworths  [edit]
   
   ✓ Looks right     ✏️ Let me fix this
   ```
7. User confirms or edits → values populate the expense form
8. Receipt photo auto-attached as a document

### Parsing Logic (SA-specific patterns)
```javascript
function parseReceiptText(text) {
  let result = { amount: null, date: null, vendor: null };
  
  // AMOUNT: Look for total patterns (SA Rand)
  // Priority order: TOTAL > AMOUNT DUE > BALANCE > largest R amount
  const totalPatterns = [
    /(?:TOTAL|TOTAAL|AMOUNT\s*DUE|BALANCE|BAL)\s*R?\s*(\d[\d\s]*[.,]\d{2})/i,
    /R\s*(\d[\d\s]*[.,]\d{2})/g  // fallback: any R amount
  ];
  
  // Try specific patterns first
  for (const pattern of totalPatterns) {
    const match = text.match(pattern);
    if (match) {
      result.amount = parseFloat(match[1].replace(/\s/g, '').replace(',', '.'));
      break;
    }
  }
  
  // If no specific match, find the largest R amount (likely the total)
  if (!result.amount) {
    const allAmounts = [...text.matchAll(/R\s?(\d+[.,]\d{2})/g)];
    if (allAmounts.length) {
      result.amount = Math.max(...allAmounts.map(m => 
        parseFloat(m[1].replace(',', '.'))
      ));
    }
  }
  
  // DATE: SA formats
  const datePatterns = [
    /(\d{1,2})[\/\-.](\d{1,2})[\/\-.](\d{4})/,   // DD/MM/YYYY
    /(\d{1,2})[\/\-.](\d{1,2})[\/\-.](\d{2})/,    // DD/MM/YY
    /(\d{1,2})\s+(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)\w*\s+(\d{4})/i
  ];
  
  for (const pattern of datePatterns) {
    const match = text.match(pattern);
    if (match) {
      // Parse based on format found
      result.date = parseMatchedDate(match);
      break;
    }
  }
  
  // VENDOR: First meaningful line of text (skip empty lines, numbers-only lines)
  const lines = text.split('\n').map(l => l.trim()).filter(l => l.length > 2);
  const vendorLine = lines.find(line => 
    /[a-zA-Z]{3,}/.test(line) && // Has at least 3 letters
    !/^(TAX|VAT|TOTAL|DATE|TIME|CASH|CARD|CHANGE)/i.test(line)
  );
  if (vendorLine) {
    result.vendor = vendorLine.substring(0, 40); // Cap at 40 chars
  }
  
  // Known SA retailers for better matching
  const knownVendors = [
    'Woolworths', 'Pick n Pay', 'Checkers', 'Shoprite', 'Spar',
    'Clicks', 'Dis-Chem', 'Makro', 'Game', 'Builders',
    'Shell', 'Engen', 'BP', 'Sasol', 'Caltex', 'TotalEnergies',
    'Takealot', 'Mr Price', 'Incredible Connection',
    'Nando\'s', 'Steers', 'KFC', 'McDonald\'s', 'Wimpy',
    'Uber', 'Bolt'
  ];
  
  for (const vendor of knownVendors) {
    if (text.toLowerCase().includes(vendor.toLowerCase())) {
      result.vendor = vendor;
      break;
    }
  }
  
  return result;
}
```

### Accuracy Expectations
- Amount: ~80% accurate (totals are usually clear on receipts)
- Date: ~70% accurate (format varies by retailer)
- Vendor: ~60% accurate (logos and headers are hard to OCR)
- The user ALWAYS confirms — OCR is a time-saver, not a replacement for input

### Performance
- Tesseract.js first load: ~2-4 seconds (downloads language data)
- Subsequent OCR: ~1-3 seconds per image
- Show progress: "Scanning... 40%... 80%... Done!"
- Cache the Tesseract worker — don't reinitialise for every receipt

The existing Log Expense flow gets an enhanced receipt section:

The confirmed values auto-fill the expense form fields. User still selects category and work/personal.

## FEATURE 4: CROSS-DEVICE SYNC (Export/Import)

### What it does
Users can export ALL their data from one device and import it on another. This lets someone log expenses on their phone during the day and sit at their laptop in the evening to upload documents and generate reports.

### Export Flow
1. User goes to **Settings → Data → Export Data**
2. App packages ALL localStorage data into a single JSON file
3. Options to share:
   - **📥 Download File** — saves `TaxTjommie_backup_2026-02-27.json`
   - **📋 Copy to Clipboard** — for quick paste (only if data is small)
4. File includes a metadata wrapper:

```json
{
  "app": "Tax Tjommie",
  "version": "3.0",
  "exportDate": "2026-02-27T14:30:00Z",
  "exportDevice": "iPhone (mobile)",
  "dataSize": "2.4MB",
  "taxYears": ["2025/26", "2024/25"],
  "expenseCount": 147,
  "documentCount": 42,
  "data": {
    // ... entire localStorage object
  }
}
```

### Import Flow
1. User goes to **Settings → Data → Import Data**
2. Options:
   - **📤 Choose File** — file picker for .json
   - **📋 Paste from Clipboard**
   - On desktop: **drag-and-drop** zone
3. App reads the file and validates:
   - Is it a valid Tax Tjommie export? (check `app` field)
   - Version compatible? (check `version`)
   - Show summary before applying:
   ```
   IMPORT PREVIEW
   ──────────────
   From: iPhone (mobile)
   Exported: 27 Feb 2026, 14:30
   
   Contains:
   - User profile: [Name]
   - Tax years: 2025/26, 2024/25
   - Expenses: 147
   - Documents: 42 (2.4MB)
   - Budget data: ✓
   - Backpocket goals: 3
   
   ⚠️ This will REPLACE all data on this device.
   
   [Cancel]  [Import & Replace]
   ```
4. On confirm: replace entire localStorage with imported data
5. App reloads with imported data
6. Success message: "✓ Data imported. 147 expenses, 42 documents loaded."

### Merge vs Replace
For MVP, **import replaces everything**. This is simpler and safer than trying to merge:
- Merge logic is complex (duplicate detection, conflict resolution)
- Most users will have one "primary" device and sync one-way
- The replace approach is predictable — no surprise duplicates

Show a clear warning: "This will replace ALL data on this device. Make sure you've exported from this device first if you have data here you want to keep."

### Sync Workflow (Explained to User)
On the Settings → Data screen, show a simple guide:

```
SYNC BETWEEN DEVICES
─────────────────────
Tax Tjommie stores everything on your device. 
To use on multiple devices:

1. On your MAIN device → tap "Export Data"
2. Send the file to your other device 
   (email, WhatsApp, AirDrop, Google Drive)
3. On your OTHER device → tap "Import Data"
4. Select the file → done!

💡 Tip: Export regularly to keep both devices 
   in sync. Always export from whichever device 
   has the latest data.
```

### Auto-Backup Reminder
- If user hasn't exported in 30+ days and has 20+ expenses: show nudge on dashboard
- "You have 85 expenses and no backup. Export your data to keep it safe."
- Settings shows: "Last export: 14 Feb 2026 (13 days ago)"

### File Size Considerations
- Without documents: export will be <100KB (just JSON text)
- With documents (base64 images): could be 2-5MB
- Most email/WhatsApp can handle 5MB files
- If file exceeds 5MB, show warning: "Your backup is [X]MB. You may need to use a file sharing service like Google Drive."

### Data Integrity
- On export: calculate a simple checksum (e.g., expense count + document count + total amounts)
- On import: verify checksum matches
- If mismatch: "⚠️ This file may be corrupted. Import anyway? [Cancel] [Import]"
